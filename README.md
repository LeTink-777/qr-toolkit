# QR Toolkit

Мобильное приложение для сканирования и генерации QR-кодов, построенное на Expo и React Native с TypeScript.

## Возможности

- **Сканер** — сканирование QR-кодов через камеру с автоопределением типа (URL, Wi-Fi, vCard, Email, телефон, текст)
- **Генератор** — создание QR-кодов с кастомизацией внешнего вида и экспортом
- **История** — SQLite-хранилище всех сканирований и генераций с поиском и фильтрами
- **Безопасность** — проверка URL на фишинг (punycode, IP-адреса, шортнеры), запрет скриншотов для Wi-Fi паролей
- **Локализация** — русский и английский языки

## Архитектура: Feature-Sliced Design

```
src/
├── app/              # Точка входа, навигация, провайдеры, конфиг
├── features/         # Бизнес-фичи (изолированные)
│   ├── scanner/      # Сканирование QR-кодов
│   ├── generator/    # Генерация QR-кодов
│   ├── history/      # История сканирований
│   └── settings/     # Настройки приложения
├── entities/         # Доменные сущности (типы, Zod-схемы)
│   ├── scan-record/
│   └── qr-payload/
├── shared/           # Переиспользуемый код
│   ├── ui/           # Дизайн-система
│   ├── lib/          # Утилиты, хуки
│   ├── theme/        # Цвета, типографика, отступы
│   └── i18n/         # Локализация
└── infrastructure/   # Внешние зависимости
    ├── storage/      # SQLite + миграции + репозиторий
    ├── crypto/       # Хеширование, SecureStore
    └── permissions/  # Разрешения камеры
```

### Ключевые принципы

1. **Изоляция фич** — фичи не импортируют друг друга, общение через `shared` и `entities`
2. **Public API** — каждая фича экспортирует только нужное через `index.ts`
3. **Result-паттерн** — все операции с внешними системами (БД, FS, камера) возвращают `Result<T, E>`
4. **Zod-валидация** — весь внешний ввод (QR-пейлоады, формы, данные из БД, env) валидируется через Zod
5. **Безопасность по умолчанию** — приватность включена, URL проверяются перед открытием

## Как добавить новую фичу

1. Создайте папку в `src/features/new-feature/` со структурой:
   ```
   new-feature/
   ├── ui/          # React-компоненты
   ├── model/       # Хуки, Zustand-стор, типы
   ├── lib/         # Чистые функции, утилиты
   └── index.ts     # Public API
   ```

2. Добавьте экран в `TabNavigator.tsx` или создайте отдельный стек

3. Добавьте переводы в `shared/i18n/locales/ru.json` и `en.json`

4. Напишите тесты в `__tests__/unit/features/new-feature/`

## Требования к безопасности

- Все SQL-запросы параметризованы (без template literals)
- URL проверяются на punycode, IP-адреса, шортнеры
- Пароли Wi-Fi не отображаются на экранах при запрете скриншотов
- Env-переменные валидируются Zod при старте
- Чувствительные поля фильтруются в логгере
- `any` запрещён в tsconfig strict-режиме
